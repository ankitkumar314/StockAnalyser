```sql
  answer1 text null,

```

### Table: `concall_transcript`
```sql
CREATE TABLE public.concall_transcript (
  id bigint generated by default as identity not null,
  created_at timestamp with time zone not null default now(),
  transcript_url character varying null,
  quarter_date date null,
  status character varying null,
  summary_id bigint null,
  stock_id integer null,
  constraint concall_transcript_pkey primary key (id),
  constraint concall_transcript_stock_id_fkey foreign key (stock_id) references stocks (id),
  constraint concall_transcript_summary_id_fkey foreign key (summary_id) references concall_summary (id)
);
```

## 🔄 Complete Workflow

### Step 1: Create Transcript from Concall URL
```bash
POST /transcripts
{
  "ticker": "RELIANCE",
  "transcript_url": "https://example.com/q3-2024-concall.pdf",
  "quarter_date": "2024-09-30"  # Optional, auto-calculated
}
```

**Response:**
```json
{
  "id": 1,
  "stock_id": 1,
  "quarter_date": "2024-09-30",
  "transcript_url": "https://example.com/q3-2024-concall.pdf",
  "status": "pending",
  "summary_id": null,
  "created_at": "2024-04-20T10:15:30"
}
```

### Step 2: Generate Summary (Background)
```bash
POST /transcripts/generate-summary
{
  "transcript_id": 1
}
```

**Background Process (Reuses Existing AgentController):**
1. Ingests PDF using `AgentController.ingest_pdf()` → Gets `document_id`
2. Calls `AgentController.batch_evaluate()` with 5 predefined questions
3. Creates `concall_summary` record with 5 answers
4. Updates `concall_transcript` with `summary_id` and `status='completed'`

**Note:** This leverages the existing RAG infrastructure instead of duplicating code!

### Step 3: Retrieve Summary
```bash
GET /transcripts/summary/RELIANCE
```

**Response:**

## 📡 API Endpoints

| Endpoint | Method | Description |
|----------|--------|-------------|
| `/transcripts` | POST | Create transcript (quarter deduplication) |
| `/transcripts/{id}` | GET | Get transcript by ID |
| `/transcripts/ticker/{ticker}` | GET | Get all transcripts for ticker |
| `/transcripts/generate-summary` | POST | Generate summary (background) |
| `/transcripts/summary/{ticker}` | GET | Get latest summary |
| `/transcripts/batch-process` | POST | Process pending transcripts |

## 📊 5 Predefined Questions (from AgentController.batch_evaluate)

The system uses the existing `batch_evaluate` method which asks these 5 questions:

1. **answer1**: What are the key business performance highlights discussed in the call, including growth trends, revenue drivers, profitability, and any major changes compared to previous periods?

2. **answer2**: What are the most important insights and takeaways from management commentary, including strategic priorities, operational improvements, and key drivers of performance?

3. **answer3**: What is the company's future outlook as discussed in the call, including growth expectations, expansion plans, new initiatives, and key growth drivers?

4. **answer4**: What risks, challenges, or uncertainties did management highlight, including market conditions, cost pressures, regulatory risks, or operational constraints?

5. **answer5**: What major announcements, strategic initiatives, or upcoming catalysts were discussed that could significantly impact the company's future performance?
